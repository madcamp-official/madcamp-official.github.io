공통과제 III : Build Deep or Spread Wide (3~4인 1팀)

개발 기간 2026.07.16 ~ 상태 진행중

3주차는 아직 진행중이라 팀별 프로젝트 목록은 완료 후 업데이트됩니다. 아래는 이번 주 과제 주제입니다.

과제 배경

구현 자체가 빨라진 Software 3.0 환경에서는 두 가지 질문이 중요해집니다. 얼마나 깊이 있는 시스템을 만들 수 있는가, 그리고 누가 이 산출물을 실제로 사용하게 할 것인가입니다. 3주차에는 이 두 방향 중 하나를 선택해 일주일 동안 집중적으로 실험합니다. 기능을 빠짐없이 갖춘 서비스를 완성하는 것보다, 하나의 핵심 문제를 끝까지 밀어붙여 보는 실험에 가깝습니다.

옵션 1 — Build the Core

가장 어려운 핵심 문제를 깊이 해결하기. UI에 들이는 시간을 최소로 줄이고, 팀 전체가 백엔드·모델·알고리즘·프로토콜·실시간 처리 등 가장 어려운 기술 문제에 집중합니다. 산출물의 사용자가 사람이 아니라 다른 프로그램이나 agent여도 됩니다(API, CLI, SDK, MCP 등). 예시로 리누스 토르발스의 초기 Git(UI 없이 견고한 해시 기반 코어 구조)과, 실시간 음성 워터마킹·검출 시스템이었던 이전 기수 프로젝트 CallCops가 제시되었습니다. 결과는 처리 속도·정확도·오류율 등 핵심 문제와 직결된 지표 1~2개로 확인합니다.

옵션 2 — Launch & Spread

실제로 공개하고 사용자 반응 확인하기. 공개 배포가 선택이 아닌 기본 조건이며, 실제 사용자가 산출물을 쓰게 하고 그 반응을 바탕으로 기능이나 흐름을 최소 한 번 이상 수정하는 것이 핵심입니다. 완성 후 공개할 필요는 없고 랜딩페이지·프로토타입·초기 버전을 먼저 공개해도 됩니다(예: Wordle의 결과 공유 이모지, 토스의 초기 랜딩페이지). SNS·GitHub·Hacker News·GeekNews·Discord·커뮤니티 등 목표 사용자가 있는 채널에 직접 소개하고, 노출·방문·사용·반응·확산·유지 중 의미 있는 지표 1~2개로 반응을 확인합니다.

제출할 것

  1. 선택한 옵션
  2. 1주일 동안의 진행(및 몰입) 과정과 실험 결과 — 목표, 시도한 내용, 주요 결과, 예상과 달랐던 점과 실패한 시도 포함
  3. 직접 실행하거나 사용해 볼 수 있는 산출물
  4. 프로젝트 설명과 실행 방법이 작성된 GitHub 저장소
  5. 팀원별 참여 내용과 주요 기여
